# Kokpit — Build & Cleanup Howto

## Rebuild the backend binary

Run this whenever you change anything in `backend/`:

```zsh
cd ~/Code/kokpit/backend
source ~/Code/kokpit-env/bin/activate
pyinstaller --onefile --name kokpit-backend run.py

# Copy to Tauri sidecar location
cp dist/kokpit-backend ../src-tauri/binaries/kokpit-backend-aarch64-apple-darwin
```

## Rebuild the Tauri app

Run this after changing the frontend, backend binary, or Tauri config:

```zsh
cd ~/Code/kokpit
source ~/.cargo/env
cargo tauri build

# Install to Applications
rm -rf /Applications/kokpit.app
cp -r src-tauri/target/release/bundle/macos/kokpit.app /Applications/
```

## Clean up PyInstaller artifacts

Safe to delete anytime — regenerated by the build step above:

```zsh
cd ~/Code/kokpit/backend
rm -rf dist/ build/ *.spec
find . -type d -name __pycache__ | xargs rm -rf
```

## Clean up Tauri build artifacts

The `target/` folder is large (~1GB). Safe to delete, Cargo will rebuild:

```zsh
cd ~/Code/kokpit/src-tauri
cargo clean
```

## Full clean rebuild from scratch

```zsh
cd ~/Code/kokpit

# Clean frontend
rm -rf frontend/node_modules frontend/dist
cd frontend && npm install && cd ..

# Clean backend artifacts
cd backend && rm -rf dist/ build/ *.spec && cd ..

# Rebuild backend binary
cd backend
source ~/Code/kokpit-env/bin/activate
pyinstaller --onefile --name kokpit-backend run.py
cp dist/kokpit-backend ../src-tauri/binaries/kokpit-backend-aarch64-apple-darwin
cd ..

# Rebuild app
source ~/.cargo/env
cargo tauri build

# Install
rm -rf /Applications/kokpit.app
cp -r src-tauri/target/release/bundle/macos/kokpit.app /Applications/
```

## Running in dev mode (no build needed)

Three terminals:

**Terminal 1 — Ollama (if not already running):**
```zsh
ollama serve
```

**Terminal 2 — Backend:**
```zsh
source ~/Code/kokpit-env/bin/activate
cd ~/Code/kokpit/backend
uvicorn main:app --reload
```

**Terminal 3 — Frontend:**
```zsh
cd ~/Code/kokpit/frontend
npm run dev
```

Then open http://localhost:5173

## Ollama model management

```zsh
# List installed models
ollama list

# Pull a new model
ollama pull llama3.2
ollama pull codellama
ollama pull mistral

# Remove a model
ollama rm codellama

# Check memory usage
ps aux | grep ollama | grep -v grep
```

## Notes

- The backend binary (`src-tauri/binaries/`) is **not tracked by git** — rebuild it after cloning
- The `kokpit-env` virtualenv is **not tracked by git** — recreate with `python3.12 -m venv ~/Code/kokpit-env`
- Ollama runs as a background service and survives app restarts
- SQLite database is stored inside the app's data directory and persists across rebuilds
